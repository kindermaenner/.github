# kindermaenner

We build software to solve real problems in our daily lives.

This is not a collection of experiments, tutorials, or “learning projects”.
Every repository here exists because we needed a solution – and ended up building it ourselves.

## Our approach

* Solve real problems instead of creating artificial ones
* Build small, focused tools instead of large systems
* Prefer independence over external dependencies
* Use what we build in real life

## Selected projects

### Monatsblitz

A complete workflow for managing and publishing blitz chess tournaments.

* Mobile app for tournament management (round-robin, optional return matches)
* WordPress plugin for data storage and yearly statistics
* Automatic generation of formatted tournament reports

Built to support our regular blitz chess evenings from data entry to published results.

---

### Play my music

A simple alternative for music-based games without relying on streaming subscriptions.

* Scan QR codes from physical cards
* Redirect to configurable music sources
* Designed to work with locally stored music instead of external platforms

Built because we didn’t want to depend on streaming services for a single use case.

---

### Vincent

An early communication tool (pre-2000) to keep students connected while working in a company environment.

Built at a time when off-the-shelf solutions for this kind of communication were not readily available.

---

## Philosophy

We believe that real-world problems are the best drivers for learning and good software design.

No playgrounds. No artificial exercises.
Just tools that we actually use.
